# Arquitectura De Producto

La plataforma existe para operar y comercializar instrumentos diagnósticos.

No debe convertirse en un constructor genérico de encuestas.

## Objetos Centrales

- Institución
- Usuario
- Rol
- Instrumento
- Versión de instrumento
- Dimensión
- Pregunta
- Respuesta
- Cohorte
- Aplicación
- Puntuación
- Perfil
- Recomendación
- Reporte
- Exportación

## Capacidades De Plataforma

Capacidades mínimas:

- catálogo de instrumentos;
- cuenta institucional;
- configuración de cohortes;
- activación de instrumentos;
- flujo de aplicación para estudiantes;
- captura de respuestas;
- motor de puntuación;
- generación de reportes;
- tablero;
- exportación;
- guía metodológica;
- acceso por roles.

Capacidades posteriores:

- acompañante inteligente sobre resultados;
- comparación longitudinal;
- comparación referencial;
- integraciones;
- módulos institucionales personalizados;
- marca blanca;
- mercado interno de instrumentos.

## Principio De Producto

La plataforma debe formarse a partir de las necesidades reales de los instrumentos.

No construir funciones de plataforma que no mejoren directamente la aplicación, interpretación, comercialización o toma de decisiones institucionales alrededor de los instrumentos.
