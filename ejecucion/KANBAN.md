# Kanban De Ejecución

Este Kanban organiza el avance hacia el MVP de PRISMA Permanencia.

## Columnas

- **Backlog**: tareas importantes, aún no priorizadas para ejecución inmediata.
- **Por Hacer**: tareas priorizadas para el próximo ciclo de trabajo.
- **En Curso**: tareas activas.
- **En Revisión**: tareas que requieren lectura del equipo.
- **Hecho**: tareas completadas y registradas.

## Regla De Movimiento

Una tarea solo pasa a **Hecho** cuando:

- tiene archivo o resultado concreto;
- está vinculada al MVP;
- puede ser leída por el equipo;
- no depende de explicación oral para entenderse.
