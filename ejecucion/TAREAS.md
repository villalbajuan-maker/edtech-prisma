# Tareas Del MVP

Este documento convierte la hoja de ruta en tareas concretas.

## Objetivo Del MVP

Lograr que una institución de educación superior pueda aplicar PRISMA Permanencia a una cohorte y recibir resultados interpretables para priorizar acompañamiento.

## Bloque 1: Fundamento Y Alineación

Estado: en curso.

Tareas:

- crear entrada de onboarding del repo;
- documentar equipo fundacional;
- definir guía de colaboración;
- organizar Kanban;
- definir etiquetas;
- conectar documentos principales mediante links internos.

Resultado esperado:

Un integrante nuevo puede entrar al repo y entender qué es PRISMA, qué estamos construyendo y dónde aportar.

## Bloque 2: Instrumento

Estado: por hacer.

Tareas:

- construir banco de preguntas V1;
- mapear cada pregunta a dimensión y subdimensión;
- definir dirección de puntuación;
- identificar preguntas de alerta crítica;
- separar preguntas diagnósticas de variables institucionales;
- revisar lenguaje responsable.

Resultado esperado:

Primer instrumento aplicable a estudiantes de ingreso.

## Bloque 3: Puntuación Y Perfiles

Estado: iniciado.

Tareas:

- ampliar modelo de puntuación;
- definir umbrales iniciales;
- validar perfiles de salida;
- ajustar reglas de combinación;
- definir prioridad de acompañamiento;
- conectar perfil con recomendación.

Resultado esperado:

Cada respuesta puede convertirse en dimensión, perfil, prioridad y ruta sugerida.

## Bloque 4: Reportes

Estado: por hacer.

Tareas:

- diseñar reporte ejecutivo;
- diseñar reporte operativo;
- definir exportación priorizada;
- definir campos mínimos por estudiante;
- definir visualizaciones necesarias;
- definir lenguaje no estigmatizante.

Resultado esperado:

La institución puede leer resultados y actuar.

## Bloque 5: Comercial Y Validación

Estado: por hacer.

Tareas:

- crear one-pager de PRISMA Permanencia;
- crear guion de conversación con IES;
- crear propuesta piloto;
- definir criterios de piloto;
- preparar manejo de objeciones;
- listar instituciones potenciales.

Resultado esperado:

Leonardo y el equipo pueden abrir conversaciones con tomadores de decisión.

## Bloque 6: Plataforma

Estado: pendiente.

Tareas:

- definir flujo mínimo de aplicación;
- definir modelo de datos del instrumento;
- definir motor de puntuación mínimo;
- definir generación de reportes;
- definir roles de usuario;
- definir manejo de privacidad.

Resultado esperado:

Base suficiente para construir el MVP tecnológico sin sobrediseñar.
