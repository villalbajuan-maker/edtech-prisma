# Etiquetas De Trabajo

Estas etiquetas sirven para clasificar tareas, issues, revisiones y decisiones.

## Por Área

- `estrategia`: categoría, tesis, posicionamiento, decisiones de negocio.
- `problema`: hipótesis de problema, microproblemas, dolor económico.
- `instrumento`: diseño del instrumento PRISMA Permanencia.
- `metodología`: dimensiones, subdimensiones, ítems, lógica diagnóstica.
- `puntuación`: scoring, perfiles, reglas de combinación.
- `reportes`: salidas ejecutivas, operativas y exportaciones.
- `comercial`: oferta, piloto, pricing, one-pager, objeciones.
- `validación`: conversaciones con IES, feedback, aprendizaje de mercado.
- `plataforma`: arquitectura tecnológica, datos, roles, motor de reportes.
- `gobernanza`: privacidad, ética, trazabilidad, estándares.
- `repo`: organización del repositorio y documentación interna.

## Por Prioridad

- `P0`: bloquea el MVP.
- `P1`: necesario para salir a cliente.
- `P2`: importante, pero puede esperar.
- `P3`: exploratorio o futuro.

## Por Estado

- `backlog`: todavía no priorizado.
- `por-hacer`: listo para iniciar.
- `en-curso`: alguien lo está trabajando.
- `revision`: necesita lectura del equipo.
- `hecho`: terminado y documentado.

## Por Responsable Sugerido

- `juan`: arquitectura, repositorio, producto digital, plataforma.
- `camilo`: pedagogía, metodología, producto educativo.
- `leonardo`: relaciones institucionales, validación con universidades.
- `equipo`: requiere lectura o decisión conjunta.

## Regla De Uso

Toda tarea debería tener al menos:

- una etiqueta de área;
- una etiqueta de prioridad;
- una etiqueta de responsable sugerido;
- una etiqueta de estado.
