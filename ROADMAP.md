# PRISMA Roadmap

## Phase 0: Strategic Foundation

Goal: define the category, first market, first problem field, and first instrument.

Status: in progress.

Key outputs:

- category thesis;
- problem hypothesis;
- microproblem forensics;
- solution hypothesis;
- first instrument brief;
- portfolio map;
- business model options.

## Phase 1: PRISMA Permanencia Methodology

Goal: transform the problem-solution fit into a complete instrument architecture.

Key outputs:

- final dimensions and subdimensions;
- item strategy;
- question bank V1;
- scoring rules;
- profile definitions;
- recommendation matrix;
- ethical language guide.

## Phase 2: Reporting And Institutional Outputs

Goal: define exactly what institutions receive immediately after applying the instrument.

Key outputs:

- student-level report specification;
- executive report specification;
- operational report specification;
- cohort dashboard requirements;
- export schema;
- decision workflow.

## Phase 3: Commercial Packaging

Goal: turn the instrument into a sellable offer.

Key outputs:

- licensing model;
- pricing hypotheses;
- sales narrative;
- buyer map;
- pilot package;
- objection handling;
- proposal template.

## Phase 4: Platform Requirements

Goal: define the minimum technology required to operate the instrument library.

Key outputs:

- data model;
- roles and permissions;
- scoring engine requirements;
- report generator requirements;
- institution workflow;
- instrument versioning;
- privacy and data handling model.

## Phase 5: MVP Build

Goal: build only after the instrument and commercial model are clear.

Potential outputs:

- instrument runner;
- scoring engine;
- report generator;
- institution dashboard;
- export module;
- basic catalog.
