# Platform Architecture

The platform exists to operate and commercialize diagnostic instruments.

It should not become a generic survey builder.

## Core Objects

- Institution
- User
- Role
- Instrument
- Instrument version
- Dimension
- Question
- Response
- Cohort
- Application
- Score
- Profile
- Recommendation
- Report
- Export

## Platform Capabilities

Minimum platform capabilities:

- instrument catalog;
- institution account;
- cohort setup;
- instrument activation;
- student application flow;
- response capture;
- scoring engine;
- report generation;
- dashboard;
- export;
- methodological guide;
- role-based access.

Later capabilities:

- Companion over results;
- longitudinal comparison;
- benchmark;
- integrations;
- custom institution modules;
- white label;
- instrument marketplace.

## Product Principle

The platform should be shaped by instrument needs.

Do not build platform features that do not directly improve instrument delivery, interpretation, commercialization, or institutional decision-making.
