# Análisis Forense De Microproblemas

Este archivo se enfoca en los tres microproblemas iniciales más potentes para PRISMA Permanencia.

## 1. Detección Tardía

### Hipótesis

La institución detecta el riesgo cuando el estudiante ya empezó a fallar, faltar, acumular presión financiera, aislarse o desconectarse emocionalmente.

### Falla Institucional

Muchos sistemas de alerta temprana se basan en el primer dato negativo visible:

- bajo rendimiento;
- inasistencia;
- mora financiera;
- cancelación de materias;
- bajo uso de plataformas;
- reporte docente;
- solicitud de aplazamiento o retiro.

Esas señales pueden ser tempranas desde el punto de vista administrativo, pero con frecuencia son tardías desde el punto de vista humano.

### Cadena Forense

```text
Ingreso administrativo
-> expectativas no diagnosticadas
-> choque con la vida universitaria
-> dificultad temprana silenciosa
-> bajo rendimiento o inasistencia
-> alerta institucional
-> contacto tardío
-> estudiante ya desconectado
-> retiro, pausa o permanencia frágil
```

### Qué Debe Hacer El Instrumento

El instrumento debe hacer visibles condiciones de entrada antes de que existan datos negativos formales.

Debe identificar:

- fragilidad inicial;
- desalineación entre expectativas y realidad;
- dudas sobre la elección del programa;
- hábitos de estudio débiles;
- presión financiera;
- red de apoyo limitada;
- baja disponibilidad de tiempo;
- bajo sentido de pertenencia;
- baja confianza para pedir ayuda.

## 2. Falta De Segmentación Accionable

### Hipótesis

La institución puede detectar que un estudiante está en riesgo, pero no siempre logra diferenciar qué tipo de riesgo tiene ni qué acción corresponde.

### Falla Institucional

"Estudiante en riesgo" es una categoría demasiado amplia para operar.

Estudiantes distintos necesitan intervenciones distintas:

- nivelación académica;
- orientación vocacional;
- apoyo financiero;
- planeación de carga y tiempo;
- mentoría;
- integración institucional;
- seguimiento individual.

### Cadena Forense

```text
Se detecta riesgo general
-> no se identifica la causa dominante
-> se ofrece apoyo genérico
-> el estudiante no recibe el apoyo que necesita
-> baja adherencia
-> el riesgo continúa
-> la institución concluye que el estudiante no usó el apoyo
```

### Qué Debe Hacer El Instrumento

El instrumento debe clasificar:

- nivel de prioridad;
- fragilidad dominante;
- fragilidad secundaria;
- perfil;
- ruta recomendada de acompañamiento;
- urgencia;
- dueño institucional sugerido.

## 3. Mala Asignación De Apoyos Institucionales

### Hipótesis

Las instituciones suelen tener programas de apoyo, pero carecen de un mecanismo preciso para conectar cada estudiante con el apoyo correcto, en el momento correcto y con el nivel correcto de prioridad.

### Falla Institucional

El apoyo suele distribuirse mediante campañas generales, asistencia voluntaria, remisiones docentes o alertas tardías.

Los estudiantes que más necesitan apoyo no siempre son quienes lo piden.

### Cadena Forense

```text
La institución tiene servicios de apoyo
-> no hay diagnóstico fino de necesidad
-> se ofrece apoyo genérico
-> los estudiantes adecuados no llegan
-> los equipos atienden demanda, no prioridad
-> estudiantes de alta fragilidad siguen invisibles
-> la intervención llega tarde o nunca llega
```

### Qué Debe Hacer El Instrumento

El instrumento debe operar como un motor inicial de orientación institucional:

```text
Respuesta del estudiante
-> dimensión vulnerable
-> perfil
-> prioridad
-> recomendación
-> área institucional sugerida
-> acción sugerida
-> seguimiento
```

## Implicación De Producto

PRISMA Permanencia no debe detenerse en el diagnóstico.

Debe convertir diagnóstico en:

- segmentación;
- priorización;
- orientación institucional;
- recomendación accionable;
- lenguaje responsable;
- lógica de seguimiento.
